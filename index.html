Just Get Me Started!

Problem

Working with images can get frustrating in React with slow rendering speed, lengthy in-line CSS styling, and manual image reference paths.

Most image optimizations are embedded in a framework like Next.JS, requiring significant edits to existing code base.

Basic image rendering features like default fallback image are not readily available in React.

Yikes! What's the Fix?

BetterImage

BetterImage dynamically renders compressed webp files in React.

BetterImage caches the compressed images locally, resulting in faster re-renders (It also dynamically references compressed images without needing manual imports).

BetterImage provides default placeholder image when the image fails to load as expected.

BetterImage enables declarative image styling like filters and transformations.

Results

59.8% less memory transferred over the network (Left image rendered using original images versus Right image rendered with BetterImage).

Google Lighthouse Performance score improved by 15.9% (Time to Interactive decreased by 300ms including conversion time)

Applying multiple effect on images has never been easier (version 1.0.0 supports all CSS transformation and filter effects. Refer to Readme for base units and default parameters)
